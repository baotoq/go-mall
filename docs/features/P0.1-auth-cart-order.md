# P0.1 — Auth on cart and order

## Problem

`catalog` (writes) and `payment` already require a Keycloak-issued JWT. `cart` and `order` accept any caller. Carts are keyed only by `session_id` — anyone who learns or guesses a session ID can read or mutate it. Orders are looked up by `order_id` with no ownership check. Without auth on these surfaces, basic abuse (cart hijack, order enumeration, refund fraud upstream) is trivial.

## User stories

- As a **logged-in shopper**, I want only my own cart and orders to be readable and mutable by me, so that another user can't steal or alter my contents.
- As a **platform operator**, I want every write RPC on cart and order to require a valid JWT, so that bots and unauthenticated clients can't enumerate or tamper with state.
- As a **guest shopper**, I want to keep using a session-scoped cart without logging in, but to upgrade my cart to my account on login, so that I don't lose items at the auth boundary.

## Acceptance criteria

- [ ] All `cart.*` and `order.*` write RPCs return `Unauthorized` (401 / `UNAUTHENTICATED`) when no Authorization header is present or the JWT signature is invalid.
- [ ] Read RPCs (`GetCart`, `GetOrder`, `ListOrders`) require a valid JWT and return only rows where ownership (user_id or session-bound claim) matches the JWT subject. Cross-account access returns `NotFound`, never the row.
- [ ] Guest carts remain supported on a separate, explicitly-marked path, OR are migrated to a sign-in-required flow — decision documented in this file's Notes section before merge.
- [ ] JWKS URL is fetched at startup from the Dapr secret store using `CART_KEYCLOAK_JWKS_URL` and `ORDER_KEYCLOAK_JWKS_URL`, with a `KEYCLOAK_JWKS_URL` shared fallback (mirrors `catalog`).
- [ ] Failure to fetch JWKS at startup panics with a clear message — same retry policy as the existing Dapr secret fetch (12× at 5s).
- [ ] kratos `selector.Server` middleware is wired so admin-only endpoints can be added later without re-plumbing.

## Out of scope

- Replacing or modifying Keycloak itself.
- Role-based authorization beyond owner-only checks (admin separation lives in P3.3).
- Service-to-service mTLS or short-lived service tokens.
- Migrating in-flight guest carts to authenticated users at first login (deferred until P1.1 identity ships).

## Implementation notes

### Files / packages
- `app/cart/internal/server/grpc.go`, `app/cart/internal/server/http.go`
- `app/order/internal/server/grpc.go`, `app/order/internal/server/http.go`
- `app/cart/internal/conf/conf.proto`, `app/order/internal/conf/conf.proto` — add `auth { jwks_url }` block
- `app/<svc>/cmd/server/main.go` — Dapr-backed secret fetch loop
- `app/<svc>/cmd/server/wire.go`, `wire_gen.go` — provide JWKS resolver
- Consider extracting `pkg/server/auth.go` only after both services are wired identically (don't pre-abstract).

### Pattern reference
- `app/payment/internal/server/grpc.go` and `app/payment/internal/server/http.go` already use `selector.Server(jwt.Server(...))`. Mirror exactly.
- `app/catalog/cmd/server/main.go` shows the Dapr-backed JWKS URL fetch.

### Schema changes
None. Ownership columns already exist (`session_id` on cart, `user_id` on order). Change is purely middleware-level.

### Events
- **Emits:** none new.
- **Consumes:** none new.

### API surface
No new RPCs. Existing RPCs gain auth requirement; guest path explicitly marked.

## Risks & mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Web frontend silently breaks because no JWT is sent | high | high | Land web-side auth header in same PR; smoke-test in `tilt up` before merge. |
| JWKS rotation invalidates running pods | low | high | kratos `jwt.Server` caches with refresh; verify TTL config matches Keycloak rotation. |
| Guest cart contract becomes ambiguous | med | med | Decide guest semantics in this PR; document under Notes; add regression test. |
| Performance regression from per-request JWKS verify | low | low | kratos caches public keys; benchmark p95 before/after. |

## Test plan

- **Unit:** middleware test — table-driven, rejects missing/invalid JWT, accepts valid one. `app/<svc>/internal/server/auth_test.go`.
- **Integration:** Keycloak via testcontainers; issue token, hit gRPC + HTTP servers. Cover valid, expired, signature mismatch, ownership mismatch.
- **E2E:** `app/web` Playwright login → add to cart → checkout. Verify `Authorization: Bearer …` on every cart/order call.
- **Manual smoke:** `make dev`, `POST /v1/cart/items` without token → 401, with token → 200.
